## Технический стек

Статический сайт без сборки:

```
/
├── index.html
├── css/style.css
├── js/script.js
├── data/content.ru.json
├── data/content.kk.json
└── assets/
    ├── logo.png (предоставлен пользователем)
    ├── images/ (плейсхолдеры до получения реальных фото)
    └── icons/
```

- `js/script.js` при загрузке читает `data/content.<lang>.json` и рендерит текст в элементы с атрибутом `data-i18n="key.path"`
- Язык хранится в `localStorage`, по умолчанию — русский
- Никакого build-шага — открывается напрямую в браузере или заливается на любой статический хостинг

## Структура страницы (порядок секций)

1. Header — логотип, RU/KK переключатель, телефон, sticky
2. Hero — заголовок с УТП, кнопка «Оставить заявку» (якорь на форму)
3. Направления/курсы — карточки ЕНТ / НИШ / БИЛ (классы, что входит)

5. Преимущества/УТП центра

7. Фото/видео центра (галерея, плейсхолдеры)
8. Отзывы (карточки с цитатами, плейсхолдеры)
9. Цены/тарифы (плейсхолдер-цифры + кнопка заявки)

12. Footer — адрес, телефон, соцсети, встроенная карта (2GIS/Google по адресу)

Плавающая кнопка WhatsApp закреплена во всех секциях.

## Логика формы заявки

Поля: Имя (текст, обязательное), Телефон (маска `+7 (___) ___-__-__`, обязательное), Класс/направление (select: ЕНТ 10 кл / ЕНТ 11 кл / НИШ 5 кл / НИШ 6 кл / БИЛ 5 кл / БИЛ 6 кл).

При отправке (клиентский JS, без backend):
1. Валидация полей (подсветка ошибок, кнопка неактивна до заполнения)
2. Формирование текста: `Здравствуйте! Меня зовут {имя}, интересует {направление}. Телефон: {телефон}`
3. `encodeURIComponent` → редирект на `https://wa.me/77071911372?text=...`

Персональные данные никуда не передаются и не хранятся — форма только формирует ссылку на WhatsApp-чат с предзаполненным сообщением.

## Дизайн-система

Токены по трёхслойной архитектуре (primitive → semantic → component), см. таблицу цветов ниже. Полный CSS будет сгенерирован в `css/style.css` на этапе реализации.

**Primitive:**
- Plum (бренд): `#F5EEF6` … `#5B1A5E` (primary) … `#2A0C2E`
- Amber (акцент CTA): `#FFCF8A`, `#FFA23C`, `#F5821F`
- Neutral: cream `#FDFBFA`/`#FAF5F3`, gray text `#2A2230`/`#6E6470`

**Semantic:**
- `--color-primary` = plum-600 (фон header/hero, основной бренд-цвет)
- `--color-accent` = amber-500 (только кнопки CTA «Оставить заявку», чтобы всегда выделялись на фиолетовом фоне)
- `--color-background` = cream-50, `--color-surface` = white

**Typography:** Manrope/Nunito, поддержка кириллицы и казахской латиницы/кириллицы.

**Форма/радиусы:** скруглённые карточки (`--card-radius: 1.25rem`), скруглённые CTA-кнопки (`--button-cta-radius: full`), мягкие тени.

**Визуальный мотив:** силуэт моста из логотипа как декоративный SVG в hero и как разделитель между секциями.

**Стиль:** тёплый, дружелюбный, современный (не строгий корпоративный, не premium-тёмный).

## Обработка ошибок / edge cases

- Пустые/невалидные поля формы — подсветка, кнопка отправки неактивна
- Отсутствующие фото — нейтральный плейсхолдер с иконкой вместо сломанного изображения
- Mobile-first адаптивность (основной трафик — с телефона)

## Проверка перед сдачей

- Открыть в браузере (Playwright), проверить:
  - переключение языка RU ⇄ KK
  - корректность WhatsApp-ссылки с реальным номером +77071911372
  - адаптивность на мобильном разрешении
  - работу FAQ-аккордеона и якорных ссылок навигации

## Вне рамок (не делаем сейчас)

- Backend/CRM-интеграция — не нужна, лиды идут напрямую в WhatsApp
- Личный кабинет, блог, мультистраничность — не требуется для лендинга одного филиала
- Многоязычный английский — отложен, только RU/KK
